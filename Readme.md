# HODU Landing Page (Responsive)

고양이 이미지를 활용한 **랜딩 페이지(홈페이지 메인)**를 HTML/CSS로 구현한 프로젝트입니다.  
상단 고정 헤더(Header)부터 Hero 섹션, Gallery 섹션, 구독(Subscribe) UI, 배너 이미지, 푸터(Footer)까지 하나의 흐름으로 구성되어 있으며, **데스크톱/모바일 반응형 레이아웃**을 적용했습니다.

---

## 🔗 프로젝트 개요

- **프로젝트 목적**
  - Figma 시안을 바탕으로 랜딩 페이지를 마크업하고, CSS로 레이아웃/스타일을 구현한다.
  - Flex / Grid 레이아웃을 적절히 혼합하여 섹션별 UI를 구성한다.
  - 미디어쿼리로 모바일 환경(768px 이하)에서 레이아웃이 자연스럽게 변하도록 만든다.

- **핵심 구현 포인트**
  1. `header` 상단 고정 (`position: fixed`) + z-index로 콘텐츠 위에 유지
  2. Hero 섹션: 텍스트/이미지 2컬럼 구조 → 모바일에서 세로 배치 전환
  3. Gallery 섹션:  
     - Gallery-1: 이미지/텍스트 좌우 배치 (Flex)  
     - Gallery-2: 이미지 3개를 Grid로 배치 → 모바일에서 1열로 전환  
     - Gallery-3: 구독 박스를 카드 형태로 배치(배경색/둥근 모서리/인풋 UI)
  4. 공통 컴포넌트(`.container`, `.btn`, `img`)로 스타일 재사용
  5. CSS 변수(`:root`)로 컬러/레이아웃 기준값 통합 관리

---

## 🧩 페이지 구성

### 1) Header (고정 헤더)
- 왼쪽: 로고 이미지
- 오른쪽: 내비게이션 링크(Home, Support) + Download 버튼
- 스크롤을 내려도 헤더가 화면 상단에 고정되도록 처리했습니다.

### 2) Hero Section
- 랜딩 페이지 첫 화면 영역
- 텍스트 박스(제목/설명/Download 버튼) + 이미지 박스(고양이 이미지)를 좌우로 배치
- 데스크톱: 2컬럼(Flex)
- 모바일: 세로 배치로 전환 + 텍스트 중앙 정렬

### 3) Gallery Section
#### Gallery-1
- 큰 이미지 1개 + 설명 텍스트 구조
- Flex로 좌우 정렬, 모바일에서는 세로 정렬로 변경

#### Gallery-2
- 상단: 제목(h3) + 설명 문단
- 중단: 3개 이미지 갤러리(Grid 3열)
- 하단: 설명 문단 + Learn More 버튼(가로 배치)
- 모바일에서는 Grid를 1열로 바꾸고, 버튼 영역도 세로로 변경했습니다.

#### Gallery-3 (Subscribe)
- 배경색이 다른 강조 카드 UI
- 왼쪽: 구독 안내 텍스트
- 오른쪽: 이메일 입력창 + Subscribe 버튼
- 둥근 모서리, 인풋 영역 화이트 배경으로 UI 분리

### 4) Banner Section
- 큰 배너 이미지(고양이 클로즈업)를 단독 섹션으로 배치했습니다.

### 5) Footer
- 왼쪽: 로고
- 오른쪽: SNS 아이콘 링크 4개(블로그/인스타/페이스북/유튜브)

## 🛠 사용 기술

- **HTML5**
  - 시맨틱 태그(`header`, `nav`, `main`, `section`, `footer`) 중심 구조화
- **CSS3**
  - CSS 변수(`:root`)로 컬러/컨테이너 폭 관리
  - Flexbox: 헤더/히어로/갤러리 레이아웃 구성
  - Grid: 이미지 갤러리 3열 구성
  - Media Query: `max-width: 768px` 기준 반응형 처리
- **Reset CSS**
  - 브라우저 기본 스타일 차이를 줄이기 위해 reset.css 사용

## ✅ 반응형 동작 정리

- **768px 초과(데스크톱)**
  - 헤더 네비게이션 표시
  - Hero: 텍스트/이미지 2컬럼
  - Gallery-2: 이미지 Grid 3열

- **768px 이하(모바일)**
  - 네비게이션 숨김(`.navbar { display:none; }`)
  - Hero: 세로 레이아웃 전환 + 텍스트 중앙 정렬
  - Gallery-1: 세로 전환
  - Gallery-2: 이미지 Grid 1열 전환 + 버튼 영역 세로 전환
  - 전체 폰트/간격을 모바일에 맞게 축소

## 🔍 구현 중 고민/개선 아이디어

- **Header 고정으로 인해 첫 섹션이 가려질 수 있음**
  - `main` 또는 `.hero`에 `padding-top`을 주어 헤더 높이만큼 공간 확보하면 더 안정적입니다.
- **컴포넌트 클래스 통일**
  - `.text-box`, `.text-box-1`, `.text-box-2`처럼 파편화된 네이밍을 정리하면 유지보수성이 올라갑니다.
- **접근성(Accessibility)**
  - 버튼/링크의 hover/focus 스타일을 추가하면 키보드 탐색이 더 좋아집니다.
  - input에 label 연결을 추가하면 더 표준적인 폼이 됩니다.
